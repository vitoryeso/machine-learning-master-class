  • Tier 1 — resolução exata (ex.: 3840×2160 → desktop_wallpaper)
  • Tier 2 — pasta + aspect ratio (ex.: pasta iPhone, portrait → camera_photo)
  • Tier 3 — fallback por aspect ratio genérico
  • has_people = 1 se a imagem pertence a people/
  • Split: 70/15/15, estratificado por image_type

Curvas de Treino — Loss & Métricas de Validação

MLP CLIP mantém val_loss estável (~0.5–0.7). MLP ConvNeXt apresenta overfitting acentuado: val_loss sobe de 1.0 a 1.66 enquanto o treino continua caindo.
Checkpoint salvo no melhor val type-F1 (early stop implícito).

Fine-Tuning End-to-End

Setup

  • Encoder descongelado: backbone inteiro + cabeça dual-head treinados conjuntamente
  • FT CLIP ViT-B/32: 151M params (vs 165K frozen)
  • FT ConvNeXt-Tiny: 28M params (vs 231K frozen)
  • lr=1e‑5 (10× menor), cosine annealing, 30 epochs
  • RTX 4090 — ~15 min/modelo (vs segundos no probe)

Resultado Surpreendente

  • FT CLIP type_f1 = 0.835 < frozen MLP CLIP 0.868
  • FT ConvNeXt type_f1 = 0.810 < frozen MLP ConvNeXt 0.849
  • Fine-tuning degradou a performance de classificação — overfitting em 8K imagens com 151M params
  • people_f1 empata (~0.90): task mais simples é robusta ao overfit
  • Conclusão: features pré-treinadas frozen + MLP leve é a melhor estratégia para datasets pequenos

Interpretação

Por que CLIP > ConvNeXt?

  • Pretraining contrastivo em 400M pares imagem-texto força representações semânticas ortogonais por conceito — ideal para separar categorias visualmente heterogêneas
  • ConvNeXt aprende features para 1.000 classes ImageNet; screenshot_desktop ou ai_generated não existem lá
  • Linear probe CLIP (3.591 params) ≈ MLP ConvNeXt (230K params): as features CLIP já são quase linearmente separáveis
  • ConvNeXt overfita porque 768-dim features não generalizam para domínio pessoal

Tarefas & Transferência

  • has_people empata (CLIP ≈ ConvNeXt, F1 ~0.83): presença humana é feature visual saliente para qualquer CNN profunda
  • screenshot_desktop F1=0.54 — baixo suporte (44 amostras) + alta variância visual; vai melhorar com dataset completo

Limitações

  • Labels derivados de metadados — ruído em casos ambíguos
  • has_people baseado apenas na pasta people/ — falsos negativos em camera_photo e thumbnail
  • Fine-tuning degradou type_f1 (0.835 vs 0.868) — dataset pequeno demais para 151M params
  • Classe screenshot_desktop subrepresentada (3.7%)
  • Amostra de 8.000 imagens — dataset completo (>20 K) ainda pendente

Conexão com ViT Compression

  • CLIP ViT-B/32 com frozen weights já entrega type_f1=0.869 — evidência de redundância representacional
  • Se quantização + poda (mestrado, Paper 3) comprime o encoder mantendo 85%+ das features, a degradação na probe deve ser mínima
  • Este projeto fornece um benchmark de preservação de features para avaliar o impacto da compressão em tarefas downstream fora de ImageNet
